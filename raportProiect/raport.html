1. Introduction

1.1 Purpose

Our web application aims to aid users with their to day lives when it comes to cooking and planning their weekly meals.

1.2 Scope

The application will provide users with a list of recipes based on their preferences and will also generate a shopping list based on the recipes they choose. Also, it will provide users with the possibility to save their favorite recipes and to create a list of ingredients they need for their recipes. The application will also provide users with the possibility to create an account and to log in to their account in order to save their preferences and recipes.

The application allows users to search for recipes based on the ingredients and popularity of the recipes. The application will also provide users with the possibility to filter the recipes based on the type of meal they want to cook, whether that's a breakfast , lunch or dinner.

2. Overall Description

2.1 Product Perspective

This platform is designed as a time saver for its users. It serves as a helper with the daily meal planning and shopping, providing personalized recommendations based on user preferences and project goals. The platform is intended to be user-friendly, accessible, and responsive across various devices.

2.2 Product Functions

  • Recipe saver
  • Recommandation based on popularity
  • Ingredients saver
  • Customizable User Profile

2.3 User Classes and Characteristics

Different user classes are: Authenticated Users, who have access to basic functionalities enlisted above , who enjoy personalized recommendations tailored to their User Profile and Administrators, who moderate the application by banning and unbanning users.

Each class has different functionalities.

2.4 Operating Environment

The software is designed to operate in a web-based environment, utilizing technologies such as Node.js for the backend and HTML, CSS, and JavaScript for the frontend.

2.5 Assumptions and Dependencies

Key assumptions include the accessibility to a MySQL database for the resources and various user preferences.

3. External Interface Requirements

3.1 User Interface

The platform will feature a simple, responsive web design, compatible with various devices and screen sizes (no screen layout constraints). The UI will be intuitive and user-friendly, with a focus on accessibility and ease of use. The design will be clean and modern, with a color scheme that is easy on the eyes.

Basic sketch of the UI

3.2 Hardware Interfaces

As a web-based application, there are no specific hardware interface requirements. The platform is accessible and performant on any standard device.

3.3 Software Interfaces

The backend database will be MySQL and will store various information regarding recipes, user data and preferences.

4. System Features and Requirements

4.1 Personalized Recommendations

Based on dietary restrictions and allergies, users will only view recipes that are suitable for their needs. The platform will also provide recommendations based on popularity of recipes.

4.2 Requiring certain statistics

Authenticated users have the ability to request certain statistics sent to their email adress having one of the following options:

  • Weekly spent
  • BMI calculator
  • Most popular recipes

4.3 User Preferences and Profile Management

Users can manage their profiles, including personal information, dietary restrictions, and allergies. Preferences can be updated at any time, and the platform will adjust recommendations accordingly.

4.4 Administrator Capabilities

Administrators have access to a list of users and can perform the following actions:

  • Ban
  • Unban

This way, the management is kept at a minimal level, yet an efficient one.

5. Other Nonfunctional Requirements

5.1 Performance Requirements

The platform should be responsive and performant, with fast loading times and minimal downtime. The backend should be optimized for speed and efficiency, ensuring a smooth user experience.

5.2 Security Requirements

The platform should be secure, with user data encrypted and protected. The backend should be secure against SQL injection and other common vulnerabilities. User authentication should be robust and reliable.
